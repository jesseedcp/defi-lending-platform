这份手册是给**没有代码背景的演示人员**使用的。

## 0. 先读这一段：这个项目到底是什么

这个项目不是传统网站，它没有“后端管理系统”或“账号密码登录”。

它的真实运行方式是：

1. 借贷合约已经部署在 `Sepolia Testnet`
2. 用户用浏览器里的 MetaMask 钱包连接 `Sepolia`
3. 前端页面直接读取链上数据
4. 用户的存款、借款、还款、提取、领奖励等操作，都会发成链上交易

### 通俗解释

- **Sepolia 测试网**：公开可访问的以太坊测试网络，可以理解成“线上测试环境”
- **合约地址**：这个借贷系统在区块链上的门牌号
- **钱包**：登录身份 + 签字工具
- **Chain ID**：当前连接的是哪一条链的编号
- **抵押品**：用户先放进去作为担保的资产
- **借款**：在有担保的前提下，从池子里借出另一种资产
- **Health Factor / HF（健康因子）**：风险分数，越高越安全，低于 1 代表可能被清算
- **LTV（贷款价值比）**：借款金额占抵押价值的比例
- **清算**：当风险太高时，系统允许别人替你还一部分债，并拿走一部分抵押品
- **Approve（授权）**：允许合约代你操作代币，可以理解成“先给权限，再执行转账”
- **Flash Loan（闪电贷）**：一笔必须在同一笔链上交易里借出并归还的临时贷款

### 3.9 不建议在现场临时做的事情

## 8. 模块二：Dashboard 总览

### 操作路径

`Home -> Go to Dashboard`

### 要讲什么

“这一页是总览页，可以看到用户抵押了多少、借了多少、风险是否安全、还能借多少，以及奖励积累情况。”

### 点击步骤

1. 进入 `Dashboard`
2. 先读顶部风险横幅
3. 依次介绍关键指标卡片
4. 往下看 `Protocol Summary`
5. 再看 `Your Assets`
6. 点击一次 `Refresh`

### 预期效果展示

- 若尚未借款，HF 通常较高或显示为安全
- 能看到资产列表
- 能看到供应利率、借款利率、池子流动性

### 关键数据展示

优先讲这 5 个数字：

1. `Total Collateral`：抵押总值
2. `Total Debt`：债务总值
3. `Health Factor`：是否安全
4. `Borrow Capacity`：还能借多少
5. `Accrued Rewards`：已累计奖励

### 面向非技术观众的解释话术

- `Total Collateral`：用户已经押进去作为担保的资产总值
- `Total Debt`：用户已经借出来、还没还的总额
- `HF`：风险分数，越高越安全
- `Borrow Capacity`：系统还允许继续借出的额度
- `Accrued Rewards`：因为参与协议而累计的奖励

## 19. 故障应急总表

| 场景            | 快速判断                | 快速处理                      |
| ------------- | ------------------- | ------------------------- |
| 首页打不开         | 浏览器无法访问 `localhost` | 检查前端服务终端是否还在运行            |
| 前端能开但按钮灰掉     | 钱包未连接或网络不匹配         | 连接钱包并切换到 `11155111`       |
| Dashboard 没数据 | Pool 地址错误或网络不对      | 回首页重填地址，确认当前网络是 `Sepolia` |
| 交易一直不动        | MetaMask 有未处理弹窗     | 打开 MetaMask 完成或取消上一步      |
| 奖励领取失败        | 奖励准备不足              | 让技术同事检查 GOV 准备情况          |
| 借款失败          | 池子流动性不够             | 让技术同事补充流动性                |
| 清算失败          | 仓位还不危险              | 跳过清算执行，只做讲解               |
| Flash Loan 失败 | 接收合约不完整             | 直接跳过此模块                   |

***
